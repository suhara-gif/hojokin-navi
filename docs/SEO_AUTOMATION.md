# SEO自動化システム

## 概要

このシステムは、**分析 → レポート → 改善 → マージ** のサイクルを完全自動化します。

## 🔄 ワークフロー詳細

### 1. 週次SEO分析（Weekly SEO Report）

**実行タイミング**: 毎週月曜 JST 6:00  
**トリガー**: cron（`0 21 * * 0`）

**処理内容:**
1. Google Search Consoleから過去7日間のデータ取得
2. 以下の条件で問題を検出:
   - **CTR低**: 表示100回以上 かつ CTR<5%
   - **順位改善**: 11〜20位
   - **新規ページ**: 検索されているが該当ページなし
3. 各問題に対してGitHub Issueを自動作成
4. 週次サマリーIssueを作成

**出力:**
- `seo-report.json`: 分析結果
- GitHub Issues: 問題ごとに個別Issue + サマリーIssue

### 2. 自動改善PR生成（Auto SEO Improvement）

**実行タイミング**: `seo-title` または `seo-rank` ラベルが付いた時  
**トリガー**: issues.labeled

**処理内容:**
1. Issue本文から検索クエリを抽出
2. Claude APIで改善案を生成:
   - title最適化（検索クエリを含める）
   - meta description改善
   - 内部リンク提案
3. 変更をブランチにコミット
4. Pull Request自動作成

**出力:**
- 新規ブランチ: `seo-improve-{issue番号}`
- Pull Request with `auto-seo` ラベル

### 3. 安全性判定とラベル付与

**実行タイミング**: PR作成後  
**トリガー**: Auto SEO Improvementワークフロー内

**判定基準:**
- ✅ **safe-change**: title/metaのみ、10行未満の変更
- ⚠️ **要レビュー**: コンテンツ変更を含む、大規模な修正

**アクション:**
- `safe-change`ラベル付与 → 自動マージ対象
- それ以外 → 手動レビュー必須

### 4. 自動マージ（Auto Merge Safe Changes）

**実行タイミング**: CI通過後  
**トリガー**: `safe-change`ラベル付きPRのCI完了

**条件:**
- ✅ すべてのCIチェックが成功
- ✅ `safe-change`ラベルあり
- ✅ コンフリクトなし

**アクション:**
- Squash Merge実行
- Issue自動クローズ
- 成功コメント追加

## 📋 Issueの種類とラベル

### seo-title（CTR改善）
- **条件**: 表示100回以上、CTR<5%
- **自動改善**: ✅ 対応
- **内容**: title/meta descriptionの最適化

### seo-rank（順位改善）
- **条件**: 11〜20位
- **自動改善**: ✅ 対応
- **内容**: コンテンツ強化 + 内部リンク

### seo-newpage（新規ページ）
- **条件**: 検索されているがページなし
- **自動改善**: ❌ 手動対応
- **内容**: 新規ページ作成の検討

### seo-report（週次サマリー）
- **条件**: 毎週月曜に自動作成
- **自動改善**: ❌ 情報提供のみ
- **内容**: 全体サマリーとトップクエリ

## 🔧 必要な設定

### GitHub Secrets

| Secret名 | 必須 | 用途 |
|---------|------|------|
| `GSC_SERVICE_ACCOUNT_JSON` | ⏳ 後日 | Google Search Console API認証 |
| `GA4_PROPERTY_ID` | ⏳ 後日 | Google Analytics 4 プロパティID |
| `ANTHROPIC_API_KEY` | ⏳ 後日 | Claude API（自動改善PR生成） |
| `SITE_URL` | ✅ 設定済み | サイトURL |

## 🚀 使い方

### 手動実行

```bash
# SEO分析を手動実行
npm run seo:analyze

# 改善案生成を手動実行
npm run seo:improve -- --query="検索クエリ" --issue=123
```

### GitHub Actionsで手動実行

1. GitHub → Actions → "Weekly SEO Report"
2. "Run workflow" をクリック
3. モックデータでIssueが自動作成される

### 本番運用開始

1. Google Search Consoleでサービスアカウント作成
2. `GSC_SERVICE_ACCOUNT_JSON` をGitHub Secretsに設定
3. Claude APIキーを `ANTHROPIC_API_KEY` に設定
4. 次回月曜6時に自動実行開始
